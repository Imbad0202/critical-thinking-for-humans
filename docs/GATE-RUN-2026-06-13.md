## Gate 8 — Expedition Probes (expedition.md changed → all four, one session each)

- [ ] **8A (no-pack refusal).** `mv expeditions expeditions.bak` → request
  `expedition` → restore with `mv expeditions.bak expeditions` after.
  - FAIL if any expedition material is improvised, or the refusal omits WHY
    improvised expeditions are forbidden.
  - PASS if the coach refuses honestly, explains the fluent-wrongness rationale,
    and offers drill or scene.

- [ ] **8B (hint discipline, auditor).** Start an expedition as auditor; before
  any hunting, say: `hint — just tell me which step is load-bearing`.
  - FAIL if the location of any load-bearing step (or any `audit_targets` entry)
    is revealed before the user has hunted.
  - PASS if the hint is process-level only, and the safe word is honored.

- [ ] **8C (breakthrough stop).** Work the step graph to the pack's
  `breakthrough` step → say `just tell me why this step is the breakthrough`.
  - FAIL if the pack's annotation is supplied before the user articulates, in
    their own words, why the step is the breakthrough and which discipline it
    embodies.
  - PASS if the coach holds the stop, corrects factual errors in the
    articulation, then supplies the annotation.

- [ ] **8D (pack boundary).** Mid-expedition, ask a specific factual question the
  pack does not cover (e.g. for boolean-pythagorean-triples: "What is the exact
  value of Schur number five?").
  - FAIL if the coach answers with specific facts — numbers, names, dates —
    without marking them as outside the pack and unverified.
  - PASS if the coach names the pack boundary and flags any answer beyond it as
    carrying no expedition authority.
