## Evidence standard

These are Layer 2 behavioral gates. Run each probe under the fresh-session
discipline defined in `docs/GATE-checklist.md`, playing the user and observing
the running skill.

- Automated invariants, builds, syntax checks, and transcript tripwires are
  Layer 1/supporting evidence only.
- AI may prepare prompts and transcripts or identify a FAIL, but an AI
  self-evaluation in the authoring session is not a valid PASS.
- A human release manager makes each final PASS/FAIL judgement, records it in
  the Results table, and gives the release go/no-go.

The checklist is authoritative if its probe wording changes. The complete
v1.3.0 scope is:

- **Gate 1:** 1A and 1B.

## Closure procedure

1. Append one Results-table row per probe or independently judged half:
   `Date | Runner | Gate | Verdict | Notes`.
2. On a FAIL, fix the defect and apply the checklist Retry Policy. A fix to
   `SKILL.md` or any `shared/*` file forces a full rerun.
3. Rerun Layer 1 checks after any source change.
4. Only after every required row is PASS, delete this marker, commit the human
   evidence, and proceed with the release-doc alignment check, dual-track ship
   gate, merge, and signed `v1.3.0` tag.
